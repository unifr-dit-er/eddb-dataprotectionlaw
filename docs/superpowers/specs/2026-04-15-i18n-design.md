# i18n — Français / Allemand

**Date :** 2026-04-15  
**Statut :** Approuvé

## Résumé

Ajouter la prise en charge du français et de l'allemand à l'application, sans routing par locale. La langue est stockée dans `localStorage`, et les colonnes NocoDB sont sélectionnées dynamiquement selon le suffixe de langue (`FR` / `DE`).

---

## Architecture

### Nouveaux fichiers

| Fichier | Rôle |
|---|---|
| `src/i18n/fr.ts` | Objet de traductions français (source de vérité) |
| `src/i18n/de.ts` | Objet de traductions allemand (même structure) |
| `src/i18n/index.ts` | Types `Locale`, `Translations` ; fonction `getTranslations(locale)` |
| `src/contexts/LanguageContext.tsx` | `LanguageProvider` + hook `useLanguage()` |
| `src/components/LanguageSelector.tsx` | Toggle FR \| DE dans la sidebar |

### Types exportés depuis `src/i18n/index.ts`

```ts
export type Locale = 'fr' | 'de'
export type LangSuffix = 'FR' | 'DE'
export interface Translations { /* toutes les clés UI */ }
export const getTranslations = (locale: Locale): Translations
```

### Hook `useLanguage()`

Retourne :
- `locale: Locale` — langue active
- `setLocale: (locale: Locale) => void` — met à jour le localStorage et re-rend
- `t: (key: keyof Translations) => string` — fonction de traduction
- `langSuffix: LangSuffix` — `'FR'` ou `'DE'`, pour les colonnes NocoDB

Comportement :
- Lecture initiale depuis `localStorage` (clé `'locale'`)
- Défaut : `'fr'` si absent ou invalide
- Écriture dans `localStorage` à chaque changement

### Intégration dans `layout.tsx`

`LanguageProvider` s'ajoute comme wrapper dans `src/app/layout.tsx`, aux côtés du `QueryClientProvider` existant.

## Colonnes NocoDB dynamiques

### Mappers (`src/lib/nocodb-mappers.ts`)

`mapDecision` et `mapKeyword` reçoivent un `langSuffix: LangSuffix` en paramètre :

```ts
export const mapDecision = (record: NocoDBRecord, langSuffix: LangSuffix): Decision => ({
  title:    String(record[`Description${langSuffix}`] ?? ''),
  abstract: String(record[`Abstract${langSuffix}`]   ?? ''),
  // ...
})

export const mapKeyword = (record: NocoDBRecord, langSuffix: LangSuffix): Keyword => ({
  label:    String(record[`Keyword${langSuffix}`]  ?? ''),
  category: String(record[`Category${langSuffix}`] ?? ''),
  // ...
})
```

### Hooks de données

`useDecisions` et `useKeywords` appellent `useLanguage()` et :
1. Incluent `langSuffix` dans leur `queryKey` — TanStack Query recharge automatiquement lors d'un changement de langue
2. Passent `langSuffix` aux mappers
3. `buildQueryString` dans `useDecisions` utilise `Description${langSuffix}` pour le filtre de recherche textuelle

## Ce qui ne change pas

- Pas de routing par locale (`/fr/`, `/de/`)
- Pas de lib i18n externe
- Pas de Server Components — tout reste Client Components avec TanStack Query
- Les cantons (`src/lib/cantons.ts`) ne sont pas traduits (codes officiels suisses)
